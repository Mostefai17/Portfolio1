Featured Project

Access Control Management System

Python Django PostgreSQL Redis

A backend-focused access control project covering authentication, role-based authorization, granular permissions, and traceable audit activity.

Architecture Overview

Django service layer, PostgreSQL data model, Redis-backed sessions, and policy checks before protected actions.

Authentication Flow

Credential validation, secure token issue, session refresh, logout invalidation, and route protection.

Audit Logs

Every sensitive login, role change, permission update, and administrative action is recorded for review.

In progress Portfolio case study

Roles

4+

Admin, manager, staff, auditor

Permissions

CRUD

Per-module policy rules

Logging

Full

Traceable admin activity

The system separates authentication, authorization, account management, and audit concerns so each layer stays understandable and testable. Views validate requests, services apply business rules, and database constraints protect role and permission integrity.

  1. User submits credentials through a validated login endpoint.
  2. Django verifies the account, status, and password hash.
  3. The session or token is issued with role context and expiry rules.
  4. Protected requests pass through middleware before reaching the view.

Role Access Level Primary Permissions
Admin System-wide Manage users, roles, permissions, and logs
Manager Operational Assign roles, approve access, review team activity
Staff Scoped Use assigned modules and update owned records
Auditor Read-only Inspect activity logs and permission history

Audit records capture actor, action, target resource, timestamp, request context, and outcome. This makes access reviews practical and gives administrators a clear history of security-sensitive changes.
